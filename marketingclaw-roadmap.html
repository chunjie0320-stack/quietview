🗺️ 三阶段产品路线图
Phase 1 2026 Q1 · 1周MVP
🔥 打通「联」——跨系统指挥官
解决占时30%、提效0%的最大黑洞。自然语言驱动12个系统的 SOP，让运营不再在系统间来回跳转。
  • 自然语言驱动跨系统 Workflow 核心
    一句话触发招商→选品→制券→权限→权益完整链路；高达/玩法/羲和/投放系统统一接入
  • 对话式活动策划 AI
    给活动目标/时间/预算，AI输出活动方案草稿（玩法/会场/人群/优惠组合）
  • 高达 CLI 集成 依赖
    基于高达 CLI 一期接口：创建/复制/发布活动，替代不稳定的 CDP 浏览器自动化
  • SOP 模板库 新增
    沉淀大促/神券/会员日等常见活动的标准 SOP，一键复用历史最优路径
Phase 2 2026 Q2 · 搭建+诊断
⚙️ 覆盖「搭配诊」——全链路 Copilot
CLI 稳定后，向搭建环节延伸。智能建场 + 批量配置 + 诊断中心，让运营把80%精力从"操作"转移到"决策"。
  • 智能建场 AI
    描述会场需求→自动选模板→生成初始布局；支持7种会场模式（大促/微主题/品牌专题等）
  • 批量组件配置 核心
    批量设置组件参数/素材/交互；配置预设库（按活动类型预加载最优配置）
  • 统一诊断中心 AI
    上线前自动检查：会场完整性/组件链路/安全合规/数据埋点；从"浅巡检"升级为"深度智能诊断"
  • 资产治理 新增
    素材库整合 + AI 标签分类；复用率分析；资产推荐（根据活动类型智能推荐组件+素材）
Phase 3 2026 Q3-Q4 · 智能化
🚀 迈向「全托管」——asB 基础设施
从辅助工具进化为商家的 asB 基础设施，能接收 asC 的个性化需求，实时响应，为 Agentic Commerce 做好准备。
  • 数智驱动活动分析 AI
    活动数据自动盯盘；异常智能预警；竞品横向对比；AI 生成复盘报告+改进建议
  • 个性化营销方案生成 AI
    根据用户画像/历史行为实时生成个性化优惠方案；支持 asC 驱动的供需协商接口
  • BD/运营自助化 新增
    微主题场景下 BD 自助搭建；无需运营干预即可完成标准活动全流程；降低平台人力成本
  • asB 协议接入 战略
    商家营销能力 Agent 化；标准化 asB 接口（契约/意图/信誉）；接入 DAS 数字生命社会协议
⚡ 关键决策 & 战略押注
🔑
高达 CLI 化是质变的前提条件
没有稳定 CLI，MarketingClaw 停留在玩具阶段
  • 当前 gundam-ops 基于 CDP 浏览器自动化，本质上是"截屏+点击"，不稳定
  • CLI 一期（2026-Q1）已在开发：创建/复制/发布接口，是真正的 API 级调用
  • CLI 就绪后，MarketingClaw 从"演示品"变成"生产工具"，搭建准确率质变
🎯
「联」是破局点，不是「搭」
做功分析最重要的洞察
  • 「联」占链路30%时间，涉及12个系统，是最大痛点
  • 高达团队自评「联」的提效=0%，说明用传统产品手段无法解决
  • Agent 编排跨系统 Workflow 是唯一解——这是 MarketingClaw 的核心壁垒
🌐
Agentic Commerce 是方向，不是噱头
MarketingClaw 是 asB 基础设施的起点
  • Google A2A 协议已被 Linux Foundation 托管，50+ 企业联署，产业化在路上
  • 美团如果不做 asB 基础设施，将成为别人 asC 调度的供应商节点
  • MarketingClaw 每一个功能都要问：这能不能成为商家 asB 的组成部分？
⚠️
已知最大风险
需要提前拿答案
  • 高达 CLI 进度：Q1能否如期上线？接口覆盖度够不够？
  • 跨系统权限：12个系统的 API 调用权限能否统一授权？
  • 用户信任：运营愿意把"发布"这个动作交给 Agent 吗？

🏪

asB 的核心承载者；未来通过 MarketingClaw 把营销能力 Agent 化，接入 Agentic Commerce 生态
