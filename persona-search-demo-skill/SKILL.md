# 画像驱动的个性化搜索 Demo

## 功能说明

本 Skill 演示如何将美团 Persona 用户画像标签应用于商品搜索排序，实现"同一关键词 × 不同用户 = 不同排序结果"的个性化效果。

**适用场景**
- 向业务方演示用户画像的价值
- 验证 Persona 标签 API 的可用性
- 搭建个性化推荐 POC

---

## 核心算法逻辑

### 1. Persona 标签获取

使用 **RAG 语义搜索接口**（非关键词搜索），支持自然语言查询：

```
GET https://persona.sankuai.com/api/v2/ai/rag/dutvs/assets/searchAll
  ?query=<自然语言描述>
  &assetTypes=label
  &idType=1
```

⚠️ **注意**：`PUT /api/v2/tcs/label/list` 对自然语言查询返回空结果，必须用 RAG 接口。

### 2. 用户画像模型

每个用户由以下维度定义：

| 维度 | 说明 | 取值范围 |
|------|------|----------|
| `priceWeight` | 价格敏感度 | 0~1，越高越在意价格 |
| `healthWeight` | 健康偏好 | 0~1，越高越偏好健康食品 |
| `qualityWeight` | 品质偏好 | 0~1，越高越偏好高品质 |
| `labelQueries` | Persona 查询关键词列表 | 字符串数组 |
| `categoryBonus` | 品类偏好加成 | `{品类: 加分值}` |

**内置用户画像（4个）**：
- 🧑‍💼 **小美**（白领女性）：健康 0.8、品质 0.7、价格 0.3
- 👨‍💻 **阿强**（程序员）：价格 0.8、品质 0.5、健康 0.3
- 📚 **学妹**（在校学生）：价格 0.9、健康 0.4、品质 0.3
- 👔 **老王**（中年商务）：品质 0.8、健康 0.6、价格 0.2

### 3. 个性化评分公式

```
商品得分 = 基础得分(0.1~1.0)
         + 健康加成 × healthWeight × 0.3
         - 价格惩罚 × priceWeight × 0.25   ← 高价 × 价格敏感 = 降权
         + 品质加成 × qualityWeight × 0.2
         + 品类偏好加成
         + 随机扰动 (±0.05)
```

**商品维度定义**（每个 mock 商品携带）：
- `healthScore`：健康分 0~1
- `priceLevel`：价格档次 1(低)~3(高)
- `qualityScore`：品质分 0~1
- `category`：品类（汉堡/奶茶/沙拉/炸鸡/米饭/披萨）

### 4. 关键词匹配

搜索时对商品名称和描述做模糊匹配，命中关键词的商品进入候选集，再按个性化得分排序。

## Persona API 使用规范

### 认证方式
SSO Cookie 认证（需先在 `https://persona.sankuai.com` 完成 SSO 登录）。Demo 页面直接通过浏览器 Cookie 调用，无需额外配置。

### 工作的接口
```
GET /api/v2/ai/rag/dutvs/assets/searchAll?query=外卖偏好&assetTypes=label&idType=1
```
返回字段：`labelId`, `labelName`, `similarityScore`, `labelDesc`

### 不工作的接口（返回空结果）
```
PUT /api/v2/tcs/label/list  ← 自然语言查询无效
```
