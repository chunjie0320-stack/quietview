# 【BUG修复】定位组件吸顶与Tab组件重叠问题

## 一、需求背景

### 1.1 问题描述
在高达（美团App首页）中，定位组件和Tab组件均具备吸顶功能。当用户向下滑动页面时，两个组件同时进入吸顶状态，导致定位组件与Tab组件在视觉上产生重叠，严重影响用户浏览体验。

### 1.2 问题截图
定位组件（显示"望京恒电大厦"）在吸顶状态下，与Tab组件（"玩具童装，母婴用品"等分类标签）发生遮挡。

### 1.3 影响范围
- 影响线上所有用户的首页浏览体验
- 活动ID：1013845
- 优先级：**紧急**（线上BUG）

## 二、需求详情

### 2.1 现状分析
| 状态 | 定位组件 | Tab组件 | 表现 |
|------|---------|---------|------|
| 正常滚动（未吸顶） | 正常展示 | 正常展示 | ✅ 无问题 |
| 仅Tab吸顶 | 正常展示或已滚出视口 | 吸顶展示 | ✅ 无问题 |
| **二者同时吸顶** | 吸顶展示 | 吸顶展示 | ❌ **重叠遮挡** |

### 2.2 目标方案
**当定位组件和Tab组件同时处于吸顶状态时，隐藏定位组件，仅展示Tab组件。**

### 2.3 交互逻辑

#### 核心规则
1. 页面滚动过程中，实时检测定位组件和Tab组件的吸顶状态
2. **当二者同时吸顶时**：定位组件隐藏（display:none 或 visibility:hidden + height:0），Tab组件正常吸顶展示
3. **当Tab组件退出吸顶状态时**：定位组件恢复展示（如果此时定位组件仍在吸顶区域内）
4. 定位组件的隐藏/显示切换需要平滑过渡，避免页面跳动

#### 状态流转

```
用户向下滚动
  → 定位组件进入吸顶 → 正常展示
  → Tab组件也进入吸顶 → 检测到二者同时吸顶 → 隐藏定位组件
  
用户向上滚动
  → Tab组件退出吸顶 → 恢复定位组件展示
  → 定位组件退出吸顶 → 恢复正常布局
```

### 2.4 边界条件
1. **快速滚动场景**：需确保状态切换不闪烁，建议使用 `IntersectionObserver` 或 `scroll` 事件节流处理
2. **定位组件点击态**：如果用户正在操作定位组件（如展开定位选择面板），此时Tab吸顶不应强制隐藏定位组件
3. **不同机型适配**：需在iOS和Android上均验证吸顶状态检测的准确性

## 三、验收标准

| # | 验收项 | 预期结果 |
|---|--------|---------|
| 1 | 正常滚动（未吸顶） | 定位组件和Tab组件均正常展示 |
| 2 | 二者同时吸顶 | 定位组件隐藏，仅Tab组件吸顶展示 |
| 3 | Tab退出吸顶 | 定位组件恢复展示 |
| 4 | 快速滚动 | 无闪烁、无跳动 |
| 5 | 活动ID 1013845 | 该活动页面验证通过 |
